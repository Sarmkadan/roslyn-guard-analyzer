# Changelog

All notable changes to this project are documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [1.0.0] - 2025-09-08

### Added
- **Docker Support**: Complete Docker and docker-compose configuration for containerised analysis
- **HTML Report Generation**: HTML formatter with filterable violation tables
- **Webhook Integration**: POST analysis results to external endpoints via `WebhookHandler`
- **Incremental Analysis**: Cache-backed analysis skips files unchanged since the last run
- **Event Bus**: Publish/subscribe architecture enabling third-party extensions and monitoring
- **Background Task Queue**: Async queue for long-running or parallel analysis operations
- **Performance Metrics Middleware**: Built-in profiling collected per-run in `AnalysisStatistics`
- **Code Fix API**: `CodeFixService` providing suggested remediations for built-in rule violations
- **Makefile**: `make build`, `make test`, `make install` targets for local development

### Changed
- Middleware pipeline is now fully composable via `IMiddleware` and `AnalysisPipeline`
- CLI help text regenerated by `HelpGenerator`; error messages improved across the board
- File discovery in `AnalysisService` now runs in parallel; throughput roughly doubles on multi-core hosts
- `CacheService` supports pluggable cache key generation via `CacheKeyGenerator`

### Fixed
- Memory leak in Roslyn `SyntaxTree` lifetime during large-project scans
- Environment-variable interpolation in `ConfigurationLoader` now handles missing variables gracefully
- Exclusion-pattern matching in `AnalysisFilterBuilder` correctly handles Windows-style path separators

### Documentation
- Added `docs/getting-started.md`, `docs/architecture.md`, `docs/deployment.md`, `docs/faq.md`
- Added `docs/api-reference.md` with full interface and model documentation
- Added eight example scripts under `examples/`

## [0.9.0] - 2025-08-18

### Added
- **Dependency Injection integration**: `ServiceCollectionExtensions.AddRoslynGuardAnalyzer()` wires the full service graph
- **Result aggregation**: `ResultAggregator` merges partial results from parallel file batches
- **Data repositories**: `AnalysisResultRepository`, `ProjectRepository`, `RuleRepository` backed by in-memory storage
- **Validation service**: `ValidationService` checks rule configurations at startup

### Changed
- `RuleEngine` now resolves enabled rules dynamically from `IRuleRegistry` rather than a hard-coded list
- `FormatterRegistry` replaces the previous `switch` block; formatters self-register at startup

### Fixed
- `ConfigurationValidator` now reports all validation errors in a single pass instead of stopping at the first

## [0.8.0] - 2025-07-28

### Added
- **Middleware pipeline**: `LoggingMiddleware`, `ErrorHandlingMiddleware`, `PerformanceMetricsMiddleware`
- **`IMiddleware` interface**: uniform contract enabling custom middleware without touching core services
- **`BackgroundTaskQueue`**: thread-safe `Channel<T>`-based queue for deferred work

### Changed
- `AnalysisService` delegates cross-cutting concerns to the middleware pipeline instead of inline try/catch blocks
- Logging now consistently uses `Microsoft.Extensions.Logging.Abstractions` throughout

## [0.7.0] - 2025-07-07

### Added
- **Configuration file support**: load settings from `.roslyn-guard.json` via `ConfigurationLoader`
- **`RuleConfigurationBuilder`**: fluent API for constructing rule configurations in code
- **Exclude patterns**: glob-based file exclusion in `AnalysisFilterBuilder`
- **Log-level control**: verbosity configurable via `logLevel` in JSON config or `--verbose` flag

### Fixed
- `ConfigurationValidator` rejected valid severity strings due to a case-sensitive comparison; corrected to `OrdinalIgnoreCase`
- Rule-specific configuration blocks were silently ignored when the rule ID had trailing whitespace

## [0.6.0] - 2025-06-16

### Added
- **CSV formatter**: `CsvFormatter` produces RFC-4180-compliant output for spreadsheet consumption
- **HTML formatter**: `HtmlFormatter` renders an interactive HTML report with sortable columns
- **`FormatterRegistry`**: central registry; output format resolved by name at runtime
- **`--output` / `-o` flag**: write report to a file instead of stdout
- **Analysis statistics**: `AnalysisStatisticsService` records file counts, rule counts, and elapsed time

### Changed
- JSON report now includes `statistics` block alongside `violations`

## [0.5.0] - 2025-05-26

### Added
- **ASY001 – Async Pattern Detection**: flags `.Result`, `.Wait()`, and `Task.Run` wrapping async calls
- **NUL001 – Null Safety Validation**: reports dereferencing potentially-null values without a null check or null-coalescing operator
- **`DiagnosticsService`**: wraps Roslyn `SyntaxTree` and `SemanticModel` access for rule implementations

### Changed
- `RuleEngine` now invokes rules concurrently per file; results merged before returning
- `RuleViolation` gains a `Severity` property; violations inherit severity from their rule unless overridden

### Fixed
- `TypeNameMatcher` pattern matching returned incorrect results for generic type names with more than one type parameter

## [0.4.0] - 2025-05-05

### Added
- **CLI interface**: `CliArgumentParser`, `CliOptions`, `CommandLineProcessor`, `HelpGenerator`
- **`--format` / `-f` flag**: choose output format (`text`, `json`, `csv`, `html`)
- **`--rules` / `-r` flag**: restrict execution to a comma-separated list of rule IDs
- **`--strict` / `-s` flag**: exit with code 1 when any violation is found
- **`--quiet` / `-q` flag**: suppress console output (useful in CI pipelines)

### Changed
- `Program.cs` entry point delegates to `CommandLineProcessor`; direct `Main` logic removed

## [0.3.0] - 2025-04-14

### Added
- **LYR001 – Layer Dependency Analysis**: prevents `Repository` types from depending on `Service` or `Controller` types
- **NAM001 – Naming Convention Enforcement**: validates PascalCase for public types and methods; snake_case for private fields
- **`RuleRegistry`**: runtime registration and lookup of `AnalysisRule` implementations
- **`CodeElement` model**: carries file path, line, column, type kind, and member name into rule `ValidateAsync` calls

### Fixed
- `StringExtensions.ToPascalCase` dropped the first character on single-segment identifiers

## [0.2.0] - 2025-03-24

### Added
- **Core rule engine**: `RuleEngine` iterates registered rules and collects `RuleViolation` results
- **`AnalysisService`**: orchestrates file discovery, rule execution, and result assembly
- **`ReportingService`**: plain-text and JSON report generation
- **`OutputWriter`**: writes report to console or file path
- **`AnalysisResult` and `ViolationReport` models**
- **`JsonFormatter`**: machine-readable output for tool integration

### Changed
- Project renamed from `RoslynAnalyzer` to `RoslynGuardAnalyzer` to avoid conflicts with the official Roslyn SDK namespace

## [0.1.0] - 2025-03-03

### Added
- Initial repository setup and solution structure
- `RoslynGuardAnalyzer.csproj` targeting .NET 10.0 with `Nullable` enabled
- Core constants, enums, and extension helpers (`Constants.cs`, `Enums.cs`, `Extensions.cs`)
- `AnalysisRule` abstract base class; `RuleConfiguration` and `RuleViolation` domain models
- `PathNormalizer`, `StringExtensions`, `TypeNameMatcher` utilities
- `RoslynGuardException` hierarchy
- xUnit test project wired to the main project via `ProjectReference`
- `.editorconfig`, `.gitignore`, GitHub Actions build workflow

## Upgrade Guide

### From 0.9.0 to 1.0.0

No breaking changes. Run `dotnet tool update --global roslyn-guard-analyzer` to upgrade.

New opt-in features available in configuration:

```json
{
  "cache": { "enabled": true, "directory": ".roslyn-guard-cache" },
  "webhookConfiguration": { "enabled": true, "url": "https://your-api.example.com/analysis" }
}
```

### From 0.8.x to 0.9.0

`IRuleRegistry.GetAllRules()` now returns only enabled rules. Call `GetAllRules(includeDisabled: true)` to retrieve the full set.

### From 0.x to 1.0.0

CLI argument format changed in 0.4.0. The positional path argument replaces the old `-p` flag:

```bash
# Before 0.4.0
roslyn-guard-analyzer -p ./src

# 0.4.0 and later
roslyn-guard-analyzer ./src
```
